# CIRCULATION-SPINE-001 ¬∑ P2.2R-B1 ‚Äî Disposition

Narrowly-bounded correction of review finding **B1** (absence/unreadability conflation). No live
verification, no rail scope expansion, no backend change, no `DifferentialWorkspace.jsx` change.

## Invariant restored

```
successful read returning {run:null}   ‚Üí  "No recorded activity"   (absence, muted)
failed / unreadable request            ‚Üí  "Couldn‚Äôt read activity" (unreadable, soft-warn)
aborted / superseded fetch             ‚Üí  dropped (neither absence nor unreadable)
```

## Changes (5 files)

- **`visionActivity.js`** ‚Äî `fetchAllRuns` now resolves each operation to a discriminated
  `{ run, unreadable }` instead of a bare `run|null`: a `!res.ok` (401/403/404/500), a network
  rejection, or malformed JSON ‚Üí `{ run: null, unreadable: true }`; a successful read ‚Üí
  `{ run: <run|null>, unreadable: false }`. `AbortError` still propagates (dropped by the caller).
  `deriveEntry` gains an `unreadable` option and an early branch returning a distinct
  `present:{ key:'unreadable', label:'Couldn‚Äôt read activity', tone:'unreadable' }` with
  `isUnreadable:true, isEmpty:false`; the readable path now carries `isUnreadable:false`.
- **`useVisionActivity.js`** ‚Äî stores the `{ op: { run, unreadable } }` map (`results`); the
  active-poll check reads `r.run.status`; the **dead `error` state was removed** (per-op
  `unreadable` now carries the truth ‚Äî this was the state the review flagged as dead and part of
  the confusion). Abort still drops silently; a genuinely unexpected non-abort throw re-raises.
- **`VisionActivityRail.jsx`** ‚Äî consumes `results[op].{run,unreadable}`; `Entry` renders a
  distinct **unreadable** row ("Couldn‚Äôt read activity") before the empty row; the header summary
  distinguishes `nothing recorded yet` (all absent) from `couldn‚Äôt read activity` (all/only
  unreadable) from `N recorded ¬∑ some unreadable` (mixed) ‚Äî never labelling a failed read as
  "nothing recorded". Partial success preserved: a single failed op does not blank the others.
- **`VisionActivityRail.css`** ‚Äî a calm soft-warn (`--va-warn`) treatment for the unreadable
  status/dot/summary; not a red alarm, visibly distinct from the muted empty state.
- **`visionActivity.test.js`** ‚Äî updated `fetchAllRuns` assertions to the new shape and added a
  **`absence vs unreadability (B1)`** block: `{run:null}`‚Üíreadable-absence; 401/403/404/500‚Üí
  unreadable; network+bad-JSON‚Üíunreadable; one failed op preserves the other three; `deriveEntry`
  renders unreadable with a label that can never equal the absence label.
