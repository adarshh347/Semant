# CIRCULATION-SPINE-001 · P2.2 — Host & UX Contract

The read-only **Vision Activity** rail: the smallest honest visible form of the circulation
spine. Observation, not orchestration. Scoped to the four currently instrumented operations
(`dissect`, `refine`, `semantic_read`, `find_similar`).

## Host decision (locked, inspected)

**Host: a self-contained child at the bottom of `<aside className="diff-inspector">` in
`DifferentialWorkspace.jsx`.** Not `RegionSurface.jsx`, not `PostDetailPage.jsx`.

Inspected the rendered Differential layout (`DifferentialWorkspace.css`): `.diff-body` is a
CSS **grid** with areas `"rail stage insp"` and columns `auto minmax(0,1fr)
clamp(200px,24cqi,300px)`. The three tools (Refine/Read/Similar) already render **inside the
inspector column**; the image + tools live in the separate `stage` column.

Why this host works and cannot obscure the image tools:
- The rail is a child of the **inspector grid column**, which is structurally isolated from
  the `stage` (image) column by the grid. It therefore **cannot** overlap the image, the tool
  rail, or the stage — by construction, not by z-index luck.
- The inspector is a scrollable flex column (`gap:1rem`, `overflow-y:auto`); adding a child at
  the bottom disturbs nothing above it and needs **no** grid/CSS-Workspace edit.
- Differential already *initiates* Refine, Read and Similar and is where evidence is corrected,
  interpreted and compared — the natural place to inspect what was executed.
- Avoiding `PostDetailPage.jsx` and `RegionSurface.jsx` keeps the diff off the top-contention
  frontend files.

Mount is minimal: one import + one `<VisionActivityRail postId regions actionStatus />` before
the existing `.diff-insp-counts` footer. The three hooks (`refine`, `reading`, `similar`) are
already called in `DifferentialWorkspace`, so their `status` values feed the rail's refresh
signal with **zero hook edits**.

Rendered confirmation (fixture harness at the real inspector width, 280px and 208px): the rail
settles quietly at the bottom of the inspector column, the representative tool panel above it
is untouched, and status text wraps correctly at the narrow width. Screenshots in
`screenshots/`.

## UX contract

- **Resting = quiet & compact, collapsed by default.** One line: `VISION ACTIVITY · N recorded ▸`
  (or `nothing recorded yet`). Expanding is opt-in.
- **Each operation is a standalone block** — never a step in one invented chain, never a causal
  arrow. Order on screen is a fixed operation list, not a temporal/causal sequence.
- Per available run: operation label; status (`RUNNING`/`SUCCEEDED`/`PARTIAL`/`FAILED`/
  `CANCELLED`/`UNAVAILABLE`/stale); adapter/source; measured latency; age; an *affects* chip
  (geometry / interpretation / research); region reference (count or resolved part name);
  fallback/degraded note; and a compact `details` disclosure of stages in human language.
- **Restrained epistemic copy** (verbatim): Dissect “Visual evidence produced.”, Refine
  “Geometry revision.”, Semantic read “Interpretation only — geometry unchanged.”, Find similar
  “Research neighbours — no relation created.” Shared region ids read as “Referenced …”, never
  “caused by”/“followed from”.
- **Scope statement is explicit**: “Records dissect, refine, semantic read and find similar —
  not all vision activity.” The rail must not imply every vision endpoint is observable.
- **Visual character**: token grammar only (one `--accent`, `--line` hairlines, `--surface`
  layers, mono for latency/adapter); low-noise status dots with **no** animation; no dashboard
  cards, no terminal aesthetic, no loading theatre, no thinking animation, no causal flowchart.
  A margin recording acts of seeing.

## Honesty guardrails baked in

- No cross-run causal arrows; no temporal-proximity-as-passage language.
- Raw stored error text is **never** rendered by default (R2); only a scrubbed, opt-in
  diagnostics disclosure.
- `{run:null}` is a quiet empty state, never an error. `run_id:null` from an action means
  “activity not recorded”, not “the visual op failed”. A stale RUNNING record reads as
  interrupted, not live.
