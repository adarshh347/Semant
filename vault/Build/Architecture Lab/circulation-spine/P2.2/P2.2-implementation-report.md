## Data boundary

Uses **only** `GET /api/v1/posts/{postId}/vision-runs/latest?operation={op}` for the four
operations, in **parallel**, via the app's `API_URL` (so the config's `X-API-Key` monkey-patch
applies). The `latest` projection already embeds `events`, so the `{run_id}` endpoint is **not
needed** and not called. No timeline endpoint, no WebSocket/SSE, no Passage entity, no
`passage_id`/`parent_run_id`/cross-run dependencies, no first-class region field, no Scheduler/
Atlas/Codex, no backend model change.

**Refresh policy** (restrained, bounded): fetch once on mount / post change; refresh when a
Differential action (`refine`/`reading`/`similar` hook) transitions to a terminal status;
schedule a **single** delayed refetch (3.5 s) only while some run is `running` & not stale — it
re-arms through state and stops the moment nothing is active; plus a manual **Refresh** button.
No permanent high-frequency loop (verified: 0 polls in 6 s with no active run). In-flight
requests are aborted on post change / unmount.

## Status honesty (§5)

| State | Rendering |
|---|---|
| `{run:null}` (empty) | quiet “No recorded activity”, muted — not an error |
| RUNNING | “Observing…”, accent dot (no progress %, no animation) |
| PARTIAL | status “Partial” + a friendly line naming the degraded stage (e.g. “Evidence produced; fine decomposition was skipped.”) + the failed stage marked in the disclosure |
| FAILED | “Didn’t complete” + friendly copy; raw error never shown by default |
| CANCELLED | “Stopped” / “Stopped before finishing.” — never implies model failure |
| stale RUNNING | “Interrupted” + “This record looks interrupted — it never reported a result.”; not animated |
| `run_id:null` (action) | treated as “activity not recorded”, not a failed op (action terminal status still triggers a refresh) |
| UNAVAILABLE (refine/SAM2) | “Unavailable” / “This capability was unavailable.”; an empty refine entry stays quiet, not alarming — the rest of the rail is unaffected |

## R2 — error projection

Raw stored error text is **never** rendered by default. `friendlyMessage()` produces
curator-facing copy from operation/status/terminal_reason. Technical detail is available only
behind an opt-in `Technical detail (diagnostics)` `<details>`, and even then `scrubTechnical()`
replaces URLs → `[url]`, filesystem paths → `[path]`, tokens/long hashes → `[redacted]`, DB
URIs → `[db-uri]` (rendered-verified: a raw traceback with a cloudinary URL, `/home/...` path
and a `ghp_` token displayed as `RuntimeError at [path]: POST [url] token [redacted] failed`).
No backend error storage was changed.

## Honesty enforcement

- Each operation renders as a standalone block; no connecting lines / arrows between runs.
- Region ids surface only as co-reference (“Referenced <part>”), never linkage.
- A `hasCausalWording` guard + a rendered full-page scan confirm no causal phrasing.
- The scope line (“…not all vision activity.”) is always shown when expanded.
