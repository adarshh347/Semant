# P2.1 ‚Äî Run Correlation Investigation

Can `post_id` / `region_id` / `geometry_rev` / `input_refs` / `output_refs` correlate the
heterogeneous runs **honestly**, using only the existing P1.1 contract (no new correlation
schema)? Answer: **partially ‚Äî and the irreducible gap is reported, not invented.**

## What the contract already carries (no schema change)

| Correlator | Where it lives | Strength |
|---|---|---|
| `post_id` | first-class run field; indexed `(post_id, operation, created_at)` | **strong** ‚Äî "all runs for a post", "latest run of operation X for a post" (live-proven) |
| `operation` | first-class run field | strong ‚Äî groups by family (dissect/refine/semantic_read/find_similar) |
| `created_at` | first-class run field | ordering only ‚Äî **temporal, never causal** |
| `region_id` | `requested_profile.region_id` (refine, find_similar) + event `input_refs`/`output_refs` | **soft** ‚Äî present + queryable by scan, but *not* a first-class field and *not* indexed |
| `geometry_rev` | refine `requested_profile` + `input_refs`(base) ‚Üí `output_refs`(new) + `result_summary` | **within-run** ‚Äî links a refine's base rev to its produced rev |
| `input_refs` / `output_refs` | per-event, e.g. `{region_id, geometry_rev, space}` | observational ‚Äî "what this stage consumed/produced" |

Live evidence (this run): semantic-read carried `requested_profile.region_ids =
['seg_0','fseg_1','fseg_0','fseg_2']`; find-similar carried
`requested_profile.region_id='seg_0'` + `result_summary.space='visual_identity'`,
`neighbours=5`. So a region *is* traceable across the semantic and retrieval runs on a post ‚Äî
by shared value, on the existing fields.

## What can be honestly asserted

1. **Post-scoped grouping** ‚Äî every run on a post is found by `post_id`; per-family latest by
   `(post_id, operation)`. Fully indexed, live-verified via `get_latest_vision_run`.
2. **Region co-reference** ‚Äî runs that *mention* a region are found by scanning
   `requested_profile.region_id` / event refs. Honest as "these runs referenced region R",
   **not** as a causal chain.
3. **Within-run geometry lineage** ‚Äî a refine run's `input_refs` (base `geometry_rev`) ‚Üí
   `output_refs` (new `geometry_rev`) is a truthful, ordered, in-run fact.
4. **Temporal order** ‚Äî `created_at` orders runs; that is observation order, **not** proof
   that run B consumed run A's output.

## The irreducible correlation gap (reported, not papered over)

**Telemetry cannot prove causality *between* separate runs** without a construct P2.1 is
forbidden to add:

- Two runs sharing `post_id` + `region_id` + `geometry_rev` is **co-reference, not
  causation**. Re-runs, concurrency, and the fact that `region_id`/`geometry_rev` are
  *values* (not a unique event lineage) make shared refs necessary-but-not-sufficient for "B
  followed A because of A".
- `VisionStageEvent.dependencies[]` express causal order **within** a run (stage ids). They
  are deliberately **never** populated across runs ‚Äî doing so would invent cross-run
  causality the system did not observe.
- A semantic-read / find-similar run does **not** currently record *which* `geometry_rev` it
  read, so telemetry can't even assert "this read happened against the geometry that refine
  produced" ‚Äî only that both touched the same region at some time.
- Bridging that gap needs an explicit **cross-run correlation id / parent-run link** ‚Äî i.e. a
  correlation schema ‚Äî which is out of P2.1 scope. We do not add it, and we do not fake it.

## Honest, non-causal recommendation (NOT built here)

A future, separately-reviewed increment *could*, without claiming causality:
- promote `region_id` to an **optional first-class, indexed** run field (enables honest
  region-scoped queries instead of a `requested_profile` scan); and
- record the **read** `geometry_rev` on semantic/retrieval runs (enables an honest "was this
  run reading now-superseded geometry?" check via `geometry_rev` monotonicity).

Only a real product need should justify an explicit correlation/parent-run id. Until then the
honest posture is: **post_id/operation/region_id/geometry_rev correlate by observation;
causality between runs is not claimed.**
