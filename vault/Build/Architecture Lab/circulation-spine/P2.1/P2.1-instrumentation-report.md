## Invariants preserved (verified)

- **Authoritative geometry unchanged** — refine's mask/`geometry_rev` and the persistence
  payload are byte-identical; instrumentation only reads them for telemetry. `logic-tested`
  (`test_refine_records_run_and_preserves_geometry_and_curator`).
- **Curator state unchanged** — refine still carries `prioritised/weight/user_note/depth/
  parent_id`; semantic still `merge_curator_state`. `logic-tested`.
- **Semantic writes geometry-free** — semantic-read still `$set`s only `semantics`; every
  write asserted `{semantics}`-only. `logic-tested`
  (`test_semantic_read_records_run_and_never_touches_geometry`).
- **Retrieval writes no post state** — find-similar's post writes == `[]`. `logic-tested`.
- **Responses unchanged except optional `run_id`** — added additively to each response;
  find-similar uses `setdefault` (never overwrites the research dict). `logic-tested` + `live-verified`.
- **Write-behind** — each route uses `VisionRunRecorder` (never raises); a failing telemetry
  store yields `run_id: null`, the real write still happens, and no run doc is created.
  `logic-tested` (3 isolation tests) + `live-verified` (semantic-read).
- **Rejections stay run-free** — refine mints its run *after* `_propose_refined_region`
  (whose 400/404/503/422 guards raise first); semantic/find-similar mint after post-load.
  `live-verified` (refine 503 → no run).
- **Persist honesty (P1.1 carried over)** — `matched_count==0` → persist stage FAILED + run
  PARTIAL/`persist_no_match`; find-similar `unavailable`/`error` → PARTIAL. `logic-tested`.
- **No conflation** — operations `refine`/`semantic_read`/`find_similar` are distinct from any
  `vision_service.*` adapter name.

## Design choices (documented)

- **`enrich-regions` deliberately NOT instrumented** — it mutates `region_annotations`
  (labels) + writes the embedding sidecar, a heavier geometry-adjacent surface; `semantic-read`
  is the cleaner semantic representative. Kept out of the smallest set.
- **`refine.propose`** folds fetch + SAM2 into one stage (the shared `_propose_refined_region`
  is not refactored — geometry code stays untouched); its latency is the real measured delta.
- **Bounded cardinality** — `region_ids`/`candidate_id[]` are sliced to ≤64 to satisfy the
  P1.1 bounded-payload gate on large posts.
- **`/vision-runs/latest?operation=`** added (default `dissect`) so sibling runs are
  retrievable; `/vision-runs/{run_id}` already generic.
