# VISION-E · E0 — Embedding contract + sidecar store — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-e-embeddings`.

## Precondition — inventory (reuse, don't duplicate)

There is already ONE embedding sidecar and the project decision to keep it is LOCKED
(`architecture-lab/vision-initiative/decisions-darshan.md`): the Mongo collection
`region_embeddings` (`backend/database.py::region_embeddings_collection`), served by
`backend/services/region_embedding_service.py`, vectors stored out-of-row, each Region
carrying only an `embedding_id` pointer (`Region.embedding_id`, `extra="allow"`). FashionCLIP
(`fashion_clip_service.py`, `patrickjohncyh/fashion-clip`, 512-d) writes it; Anuraṇana
(`anuranana_service.py`) reads it via `search_similar`. The orchestrator already reserves the
DINOv2 slot (`AdapterSpec("dinov2_vits14", Capability.FEATURE, ResourceKind.GPU, deferred)`),
and `mask_geometry.mask_to_crops` already yields the identity (`alpha_crop`) and context
(`context_crop`) crops E1 needs. **E0 extends this one store; it does not create a second.**

## What E0 added (all additive)

`region_embedding_service.py` now carries the full versioned evidence-projection contract:

- **Role in the id** — `make_embedding_id(post, region, model, role=None)`. `role=None`
  returns the original `emb_{model}_{post}_{region}` **byte-for-byte** (legacy rows untouched);
  a role gives a distinct id so a region's `identity` and `context` vectors never collide.
- **Spaces** — `space_key(model, role, version, dim)` names an incomparable vector space.
  `cosine_same_space(a, space_a, b, space_b)` **raises** on a cross-space comparison rather
  than return a meaningless number — the code path that makes "never compare across
  model/version spaces" unbreakable. `search_similar(space=…)` filters to one space and routes
  every candidate through it; legacy callers keep the model-scoped path (space `None` both
  sides → comparable) with the dimension guard as backstop.
- **Versioned record** — `upsert_embedding` now stores `role`, `checkpoint`, `dim`, `space`,
  `geometry_rev`, `preprocessing_version`, `crop_version`, `source_content_hash`, `mask_hash`,
  `normalized`, `storage`, `status`, `error`, `stale_reason`, `created_at` (via `$setOnInsert`)
  — and still accepts the original minimal call unchanged.
- **Invalidation contract** — `is_stale(record, geometry_rev, source_content_hash, mask_hash,
  model, preprocessing_version)` is a pure function of the evidence axes and returns a named
  reason. A **semantic label/note edit touches none of these, so it never invalidates a visual
  embedding**; a geometry/source/mask/model/preprocessing change does. `content_hash(bytes)`
  and `mask_hash(rle)` are the stable source/mask fingerprints.
- **Indexes** — added `space` and `region_id` (retrieval is space-scoped; stale-detection /
  re-index are region-scoped), alongside the existing `post_id` / `embedding_id`.

## Gate — additive migration; legacy readable; cache/invalidation + mixed-space rejection

`backend/tests/test_vision_e0.py` (7 tests, in-memory fake collection — no Mongo):

- legacy id byte-stable; role ids distinct.
- `cosine_same_space` refuses different spaces (raises); same space scores normally.
- label edit → **not stale**; each of geometry_rev / mask / source / model / preprocessing →
  stale with the correct named reason.
- `mask_hash` changes only with the mask; `None` → `""`.
- `upsert` writes the full versioned record (role/space/hashes/geometry_rev/created_at).
- a **legacy FashionCLIP row (original fields only) stays readable and queryable** via the
  model-scoped path.
- **mixed-space search never crosses spaces** — a 384-d DINOv2 identity vector and a 512-d
  FashionCLIP vector coexist; the dino query returns only the dino row, the fashion query only
  the fashion row.

Backend suite: **168 passed** (161 + 7) — the shared service change did not regress FashionCLIP
enrichment or Anuraṇana retrieval.

**Gate result: PASS** — one store, one identity per (post, region, model, role); versioned and
self-describing; cross-space comparison impossible in code; visual embeddings survive label
edits and invalidate only on real evidence change; legacy rows fully readable.
