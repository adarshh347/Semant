# VISION-E · E1 — Canonical evidence projections — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-e-embeddings`.

## What was built

`backend/services/evidence_projection.py` — `project_region(region, image, source_content_hash)`
returns two deterministic projections of a Region's confirmed evidence, produced through the
one geometry owner (`mask_geometry`):

- **identity crop** — the exact mask composited over a neutral background (`NEUTRAL_BG`), with
  deterministic padding (`IDENTITY_PAD = 0.12` of the bbox's longer side). "What the evidence
  is", isolated. Built from a full-image alpha (`_alpha_from_rle`, numpy nearest-neighbour
  scale from the mask's own `[h,w]`; pure-Python fallback).
- **context crop** — the same evidence with controlled surrounding context
  (`CONTEXT_SCALE = 1.8`× the bbox), recording `target_mask_box` — where the mask sits inside
  the crop, in normalized coords. "What the evidence is among".

Each projection carries `source` (`mask` | `box-legacy`), a provenance dict (`crop_version`,
pad/scale, image size, `mask_hash`, `geometry_rev`, `source_content_hash`), and a
`projection_hash`.

Rules honoured: when an authoritative `mask_rle` exists it is used (never a bbox-only crop); a
Region with only a legacy `box` still projects, tagged `source="box-legacy"` so the missing
mask is explicit; the `projection_hash` is a pure function of role + crop-version + source
pixels, so a geometry change moves it and a label/note change cannot.

## Gate — deterministic; geometry invalidates both; labels do not

`backend/tests/test_vision_e1.py` (9 tests):

- both projections produced from the mask; context ≥ identity; `target_mask_box` ∈ [0,1].
- **deterministic** — repeated projection gives identical hashes and identical pixels.
- **geometry change moves both** projection hashes (mask shifted).
- **label/note/category change moves neither** — semantics are not an input.
- **mask preferred over box + isolates evidence** — a corner of the padded identity crop is the
  neutral colour, not the blue background, proving it is a mask crop, not a bbox crop.
- **box-only legacy is honest** — `source="box-legacy"` in result + provenance.
- `None` when the region has neither mask nor box.
- **hard shapes don't crash** — holes (ring), disconnected components, thin structures,
  edge-touching; each yields non-empty crops and hashes.
- **portrait and landscape** images both project within bounds.

Backend suite: **177 passed** (168 + 9).
