# VISION-E · E2 — DINOv2 shared visual representation — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-e-embeddings`.

## Model + measurements (GTX 1650, sm_75)

- **Checkpoint:** `facebook/dinov2-small` — DINOv2 **ViT-S/14**, **Apache-2.0**, hidden **384**,
  patch 14. Pure-PyTorch via `transformers` (no custom CUDA ops → no nvcc needed on cu130).
- **Precision — FP32 chosen.** On this Turing card FP16 is *slower* (warm ~57 ms vs FP32 ~14 ms)
  with no useful VRAM headroom, and FP16↔FP32 CLS agreement is **1.00000**. FP32 is faster and
  exact, at trivial cost.

| metric | value |
|--------|-------|
| whole-image encode, cold (load+run) | ~5.1 s (first call; then model resident) |
| whole-image encode, warm | **23 ms** |
| full post embed (3 regions → 7 vectors), through ModelManager | **309 ms** |
| peak VRAM (embed pass) | **107 MiB** |
| clean unload | VRAM 100 → 16 MiB, model dropped |
| patch grid / dim | 16×16 / 384 |

## What was built

- **`backend/services/dinov2_service.py`** — the encoder (lazy GPU singleton, explicit
  `unload()`). `encode_image` returns `{cls (L2), patches (16×16×384 raw), grid}` — the SHARED
  artifact. `pool_region(features, mask_rle)` coverage-weight pools the shared grid under a mask
  (returns None when the mask is sub-patch → caller crop-encodes). `encode_crop` returns a crop's
  normalized CLS. Preprocessing fixed as `dino-v1` (resize 224² square, no center-crop, so the
  grid maps linearly to normalized image coords). All vectors L2-normalized.
- **`Dinov2FeatureAdapter`** (orchestrator, `Capability.FEATURE` / `ResourceKind.GPU`,
  `license="Apache-2.0"`) — fills the roster's reserved `dinov2_vits14` slot; `load`/`unload`
  delegate to the service; `infer` returns the shared whole-image features.
- **`backend/services/evidence_embedding_service.py`** — `embed_post_regions(post, image_bytes)`:
  encodes the whole image **once** (real runs route through `ModelManager` for single-GPU
  residency + `feature_key` caching, evicting YOLO/SAM), then per Region produces an **identity**
  vector by `mask_pool` (default) and a **context** vector by `crop_cls`, plus one post-level
  `whole_image` `whole_cls` vector. Box-only / sub-patch Regions fall back to `crop_cls`. Every
  vector is written to the ONE sidecar with role, route, space, dim, checkpoint, preprocessing +
  crop version, geometry_rev, and source + mask hashes. A `route` field was added to the store
  (additive) so each vector says how it was made.

## Gate — one image, many Regions: single shared compute, deterministic, clean unload

- **Automated** (`test_vision_e2.py`, 7, fake encoder): whole image encoded **once** for 3
  regions (`whole_image_encodes==1`, `encode_image` called once — not per region); identity via
  pooling (`n_pool==3`), context via crop (`n_crop==3`); routes correct; region vectors
  deterministic and **distinct** per mask; box-only and sub-patch fall back to `crop_cls`;
  identity vs context live in **separate spaces**; unavailable encoder is explicit.
- **Live** (real DINOv2 on the Pietà): `whole_image_encodes=1` for 3 regions → 7 records, each
  correctly spaced (`dinov2_vits14|identity|dino-v1|384`, …); pooled identity deterministic and
  normalized (‖v‖=1.0); **clean unload** frees VRAM (100→16 MiB); GPU governed by ModelManager.

Backend suite: **184 passed** (177 + 7).

**Gate result: PASS** — the whole image is encoded once and reused for every Region's mask
pooling; Region vectors are deterministic, normalized, spaced and routed; the model unloads
cleanly; no duplicate feature work.

## Cache / storage
Shared features are content-cached by `feature_key(image_hash, model, checkpoint, preproc, 224)`
— re-embedding the same source is a cache hit with no recompute. Each 384-d vector ≈ 1.5 KB of
floats in the sidecar; a 3-region post = 7 vectors ≈ 10.5 KB.
