# VISION-D · D2 — Semantic pass + safe persistence — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-d-semantic`.

## What was built
- **`SemanticAnnotatorAdapter`** (orchestrator) — capability `SEMANTIC_ANNOTATE`, resource
  `REMOTE`. Runs the VLM through `ModelManager` so it gets the remote slot (concurrency 1),
  dedup, timeout, cancellation, telemetry and the content-addressed cache. Injectable
  provider (real / fake).
- **`services/semantic_pass.py`** — `run_semantic(post, image_bytes)`: builds the D1 packet,
  computes a content cache key (`semantic_key(image_hash, candidate_digest, prompt_version,
  schema_version, model)`), runs (or cache-hits) the adapter, and returns a `semantics`
  structure (assertions + relations + global + meta). `merge_curator_state` preserves
  accepted/rejected/overridden + edited labels across reruns.
- **Endpoints** — `POST /{id}/semantic-read` (persists `post.semantics` ONLY — never
  `region_annotations`) and `PATCH /{id}/semantics/{candidate_id}` (curator accept/reject/edit;
  an edit → status `overridden`, never silently replaced by a later rerun).
- **Schema** — `semantics: Optional[dict]` on `Post`/`PostUpdate` + `post_helper`.
- **Provider fix** — the response `json_schema` is now OpenAI-strict-compliant (all
  properties `required`, optionals nullable) while keeping `additionalProperties:false`
  everywhere, so geometry stays unrepresentable and the strict call is accepted.

Each stored assertion records: candidate/Region id, label + ranked alternatives, part/
material/attributes/style, confidence + uncertainty, provider/model/prompt-schema version,
created_at, status (proposed/accepted/rejected/overridden), curator_label.

## Gate — reruns change assertions only; geometry byte-for-byte unchanged

- **Live (real gpt-4o-mini) on the Pietà:** `status succeeded`, **0 dropped ids** — every
  assertion bound to a supplied candidate id: `arch_0→wall (0.85, alts background/structure)`,
  `arch_1→floor (0.8)`, `seg_0→person (0.9, alts figure/statue)`; a **global reading**
  ("centralized… sacred and contemplative") distinct from Region claims; **relations**
  citing ids (`arch_0 background-of seg_0`, `arch_1 base-of seg_0`). **No geometry** emitted.
- **Geometry invariance (byte-level):** the region_annotations sha256 is **identical**
  before and after both a failed and a successful semantic-read
  (`65fc0e86…` == `65fc0e86…`) — mask RLE, polygons, bbox and `geometry_rev` untouched; only
  `post.semantics` was written.
- **Automated:** 5 tests (`test_semantic_pass.py`) — assertions carry provenance + are
  geometry-free; the pass does not mutate geometry; RLE/polygons/bbox/`geometry_rev` invariant;
  a rerun with the same candidate set **hits the cache** (adapter called once); curator
  overrides survive a rerun. Backend **145 passed**.

**Gate result: PASS** — semantic reruns change assertions only; geometry is byte-for-byte
unchanged; assertions persist separately with full provenance and curator state; geometry
can change only through the detector/SAM/user path (this endpoint never writes geometry).

## Cache / latency
Content-addressed by `image_hash + candidate_digest + prompt_version + schema_version +
model` — a rerun with the same candidates is a cache hit (no call); a new candidate set
(a geometry change) is a new key. `force:true` bypasses the cache but preserves prior
curator state. Live call ~2–3 s; ~8.6k image tokens on gpt-4o-mini (gemini-flash-lite is the
6× cheaper fallback if cost matters).
