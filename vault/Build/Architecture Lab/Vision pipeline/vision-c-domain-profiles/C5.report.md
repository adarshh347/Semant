# VISION-C · C5 — Mixed-domain planning + profile-control UX — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-c-domain-profiles`. The final Increment C
gate: selective scheduling for mixed images + a compact profile control in the real
curator workflow, with geometry-only candidate merge and curator-data safety.

## What was built

**Backend — selective scheduling, merge safety, capabilities**
- **Selective scheduling** — `detect_regions` runs only the passes the persisted profile
  schedules (`_profile_chosen` → fashion/architecture passes; general/YOLO always runs).
  Not every model on every pixel.
- **Curator-data safety (no loss across reruns)** — creator-authored regions (`actor=creator`,
  e.g. a Refine-confirmed mask) now **survive a re-dissect intact**; auto candidates are
  merged around them, and an auto candidate that duplicates a creator region by geometry is
  dropped (the curator wins).
- **Geometry-only dedup** — `_is_duplicate_geometry` (box IoU, kind-aware): near-identical
  same-detector masks are deduped (no duplicate masks masquerading as separate evidence),
  while **a wall and a person at the same spot stay separate** (different kinds), and
  **five separate masks stay five** (no anchor prison). Never merges/parents by label.
- **`GET /vision/capabilities`** — per-adapter runtime state (ready / deferred /
  unavailable + reason) for the UI, without a model-control dashboard.

**Frontend — the compact profile control**
- `components/ProfileControl.{jsx,css}`, mounted in `RegionSurface` (the Visual pane) above
  Dissect: **Auto** (proposes a multi-label profile with confidence + reason), profile
  **override chips** (General always-on; Fashion/Architecture/Painting toggle → `PATCH`),
  and the **scheduled passes as state pills** (ready green / deferred amber / unavailable
  grey). Changing the profile re-schedules the next Dissect. Compact, theme-aware.

## Known limitations
- The router's confidence is FashionCLIP zero-shot (relative); the override exists exactly
  because Auto is imperfect. A dedicated multi-label domain classifier is a later refinement.
- Fashionpedia stays `deferred` until a serverless endpoint is set (C3).
- Dedup is box-IoU (fast); mask-IoU dedup is a straightforward tightening if needed.
