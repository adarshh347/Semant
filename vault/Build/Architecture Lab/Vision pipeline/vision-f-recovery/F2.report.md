## The unavailable → skip guard

`backend/services/vision_capabilities.py` maps each recovery action to the capabilities it needs
and resolves an action whose capability is **down** into a **planned skip** — never a partial
write. Demonstrated live: forcing SAM down resolves `rerun_profile_general` → **skip
(unavailable: sam)**. Automated (`test_vision_f2.py`, 5): available actions run; an unavailable
capability skips (sam/semantic/embed); `retain_box` never needs a capability; `plan_skips` marks
each item. Backend suite **215 passed** (210 + 5).
