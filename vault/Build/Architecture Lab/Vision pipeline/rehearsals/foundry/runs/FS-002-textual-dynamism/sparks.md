## Spark register

### spark-FS002-01 — the demonstrated live force is the question, not the source

- **Origin thread:** score §3 R1, from E1 + E3.
- **Enablement claim:** the program has two clean demonstrations that **one word changes what a model
  reports seeing** (spark-06: *address* → a face on a facade; spark-09: *motif* → shared-purpose
  talk), and **zero** instrumented runs in which a source text was present at all. "Source pressure"
  and "prompt vocabulary" have been spoken of as one thing. They are two, and only one is attested.
- **Evidence:** manifest tally, 13 runs (E1); spark-06/spark-09 in `CANDIDATE-REGISTER.md`.
- **Nearest existing Semant construct:** none, and **none is proposed** — this is a fact about how the
  program experiments, not about Semant's ontology.
- **Strength:** SPARK. The positive half is strong but **borrowed** (runs 007–009 produced it for
  other questions). The negative half is n = 11 manifests and solid.
- **What would strengthen it:** one instrumented run with a source slice, A/B'd against `absent`.
- **What would kill it:** a source-present instrumented run whose output shifts as sharply as
  spark-06's did — that would make source and prompt one force after all.
- **[SPEC] possible location:** none.

### spark-FS002-02 — `source_condition` is set to a value the runs do not have

- **Origin thread:** score §3 R3, from E1 + E2b.
- **Enablement claim:** eleven instrumented runs record `source_condition: "present"` while supplying
  zero texts. The field is **defined** as *"whether the source was there"* (`R1-schema-decisions.md`),
  so this is not a terminological quibble: any future query for "runs conducted with a source" returns
  13 and should return 2. A control variable the program believes it is recording is not recorded.
- **Evidence:** E1 table; E2b definition; run 002's own `source-notes.md` heading *"No text source"*
  contradicting its own manifest.
- **Nearest existing Semant construct:** `rehearsal-manifest.schema.json` — **research schema, not
  product schema.** Nothing about Semant's data model follows.
- **Strength:** SPARK, n = 11, machine-checkable, and it **survived its own strongest objection**
  (critique §1, raised and refuted within this run).
- **What would strengthen it:** nothing needed; it is a count.
- **What would kill it:** a definition elsewhere overriding R1's. Searched; none found.
- **[SPEC] possible location:** **the runs must not be edited.** HW-C6 settled the general form:
  *"Do not retro-fill the runs."* A validation rule for *future* manifests is the only honest option,
  and even that is not proposed here.
