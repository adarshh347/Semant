## The semantic contract — geometry is unrepresentable by construction

`backend/schemas/vision_semantic.py`:
- `SemanticResponse` / `CandidateSemantics` / `RelationProposal` / `GlobalReading` are all
  `extra="forbid"` — any `box`/`bbox`/`polygon`/`mask_rle`/`rle`/`coordinates`/`x,y,w,h`/
  `parent_id`/`geometry_rev` the model tries to emit is **rejected at parse**. Plus
  `has_geometry_key` as a legible pre-parse guard.
- **Allowed:** candidate label + ranked alternatives; part/material/attribute/style;
  confidence + uncertainty; candidate→candidate relation proposals (cite endpoints + type +
  confidence, stay proposals); image-global composition/atmosphere/colour/scene;
  `needs_better_evidence`.
- **Candidate-ID bound:** `enforce_candidate_ids` returns any id the model referenced that
  was **not supplied** — dropped by the caller, so a `person` label can never adopt a
  distant candidate it invented.
- `StoredAssertion` (for D2) wraps the interpretation with provider/model/schema version,
  created_at, and curator state — stored separately from geometry.

`backend/services/semantic_provider.py`: `SemanticProvider` (OpenRouter, gpt-4o-mini,
strict json_schema, `available()`/`state()`, drops unknown-id assertions on parse, explicit
`unavailable`/`error`/`timed_out` states) + `FakeSemanticProvider` (deterministic, id-bound).
