# VISION-E · E3 — Text-image + fashion spaces — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-e-embeddings`.

## Decisions

- **OpenCLIP (general text↔image) — DEFERRED.** The inventory found nothing planned, pinned or
  installed for OpenCLIP anywhere in the repo. The E-contract admits it "only if evaluation shows
  useful text↔evidence retrieval beyond DINOv2." Since E4 evaluation has not run, E3 defines the
  `text_image` space but keeps it **unavailable**, with an explicit reason that names the gate:
  *"OpenCLIP not installed — deferred; add only if E4 shows it beats DINOv2 on text↔evidence."*
  No premature dependency, no silent empty — a text query returns an honest unavailable state.
- **FashionCLIP (fashion) — REUSED + ACTIVATED.** The existing `fashion_clip_service`
  (`patrickjohncyh/fashion-clip`, ViT-B/32, 512-d) now represents fashion Regions in its OWN
  space `fashion-clip|fashion|vitb32|512`. FashionCLIP *represents* the garment; Fashionpedia (a
  different model) *segments/names* it — the two are never conflated or duplicated.

## What was built

- **`backend/services/retrieval_service.py`** — the space registry + router:
  - spaces: `visual_identity` / `visual_context` / `visual_whole` (DINOv2 384-d), `fashion`
    (FashionCLIP 512-d), `text_image` (OpenCLIP, deferred); availability resolved live from the
    services.
  - `route(query_kind, domain, context_sensitive)` — evidence→`visual_identity`
    (or `visual_context`); fashion domain→`fashion`; text→`fashion` (fashion domain) else
    `text_image`.
  - `find_similar(query, space, post_ids, …)` — resolves the named space and searches only it via
    `search_similar(space=…)`; an unavailable space returns `status:"unavailable"` with a reason,
    never a cross-space or silent-empty search.
  - `list_spaces()` — spaces + live availability for the curator UX / capabilities.
- **`evidence_embedding_service.embed_fashion_region`** — encodes a fashion Region's mask-isolated
  identity crop with FashionCLIP and stores it in the fashion space with full provenance
  (`route="crop_cls"`), or returns an explicit `unavailable` when FashionCLIP isn't installed.

## Gate — routing chooses the space; absent models are explicit; no cross-space cosine in code

`backend/tests/test_vision_e3.py` (9 tests):

- routing: evidence→`visual_identity`; context-sensitive→`visual_context`; fashion domain (evidence
  or text)→`fashion`; general text→`text_image` **unavailable** with a reason naming E4.
- `list_spaces` reflects availability (DINOv2 + fashion on, OpenCLIP off with reason).
- `find_similar` on an unavailable space → explicit `unavailable`, not empty.
- **no cross-space**: a DINOv2 (384) and a FashionCLIP (512) vector on one post — the fashion query
  returns only the fashion vector, the visual query only the DINOv2 vector, and
  `cosine_same_space` across the two space ids **raises**.
- FashionCLIP activation writes into `fashion-clip|fashion|vitb32|512` (never a DINOv2 space);
  unavailable FashionCLIP → explicit.

Live `list_spaces()` (real services): `visual_identity/context/whole` available, `fashion`
available, `text_image` **unavailable** (deferred). Backend suite: **193 passed** (184 + 9).

**Gate result: PASS** — query routing selects the correct space; absent/deferred models produce
explicit unavailable states; a cross-space cosine is structurally impossible (every comparison
flows through `cosine_same_space`, which raises across spaces).
