## What was built

The B4 refinement loop is now a first-class **Refine tool inside DifferentialWorkspace**,
reached through the normal curator route (`/posts/:id` → Differential), with the lab route
kept as a harness.

- **`differential/useMaskRefine.js`** — the shared hook extracted from RefineSurface: owns
  `points/box/proposal/status` + the B4 `preview`/`confirm`/`unload` endpoints. Coords are
  normalized (stage-geometry contract); the surface feeds `addPoint`/`setBoxPrompt`.
  `confirm()` returns `{region, post}`; `clear()`/`cancel` change nothing on the server;
  `release()` frees the SAM session.
- **`components/RefineSurface.jsx`** — refactored to consume the hook (RefineLab still works
  at `lab/refine/:postId` as a test harness). One implementation, two mounts.
- **`components/RegionOverlay.jsx`** — the existing registered overlay gained optional
  `proposal` (dashed evenodd mask) + `prompt` (point/box markers) + `interactive` (false
  lets refine gestures fall through to the stage). The proposal renders through the same
  `viewBox=natural` contract, so it stays registered at any size/zoom.
- **`differential/DifferentialWorkspace.jsx`** — a `refine` tool in the L5 rail:
  - **Select → Refine** primary flow: the selected part becomes the base region; refining
    upgrades its mask identity in place (`base_id`).
  - **positive point** (click), **negative point** (⇧-click), **box** (drag) prompts, via a
    `refine` branch in `onStagePointer{Down,Move,Up}` that mirrors the brush/trace
    drag-prevention (`e.preventDefault()` + pointer capture) so the image never pans.
  - proposal + prompt rendered through `RegionOverlay`; `GroundLayers` stays
    `pointer-events:none` so gestures reach the stage.
  - inspector panel with status + Clear/Cancel/Confirm; keyboard Enter=confirm, Esc=clear.
  - **Confirm** saves the new geometry revision (the endpoint persists) and refreshes the
    region **in place** via `store.updateRegion`/`addRegion` + reselect — **no reload, still
    inside Differential**. **Cancel** clears the prompt only.
  - **Auto-release**: the SAM session is released when the tool leaves `refine` and when the
    workspace unmounts.

## Regression found + fixed (why browser verification matters)

Confirming in Differential exposed a real bug the B4 lab flow had masked: the confirm
carry-over `region[k] = prev.get(k, region.get(k))` wrote **`None`** for
`prioritised/weight/user_note` when neither the base nor the new region had them, and the
`GET /posts/:id` **`response_model=Post`** then 500'd validating that region (`None` is not
a valid bool/int/str). The B4 lab only hit the append path (fields absent → defaults), so it
slipped through.

Fix (two parts + repair + test):
- `refined_mask_to_region` now sets `prioritised=False, weight=0, user_note=""` so a refined
  region is always schema-valid.
- the confirm carry-over never overwrites a valid default with `None` (coalesces).
- repaired the 3 `None` fields my test wrote into the DB (test artifact).
- **regression test** `test_refined_region_is_schema_valid` asserts `Region(**refined…)`
  validates. Backend suite **86 passed**.
