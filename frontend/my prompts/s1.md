Use the visual audit as the source of truth.

Implement only the P0 finish-up issues from the audit. Do not touch P1/P2 unless required by a P0 fix.

Main goals:

1. Flatten container stack
- Reduce repeated rounded rectangles, borders, shadows, and pill treatments.
- Keep the major split panels: Visual and Content.
- Make inner elements inside the Content panel quieter, more inline, or ghost-like.
- Avoid card-inside-card-inside-card feeling.

2. Strengthen edit-mode hierarchy
- When isEditing is active, make the Content editor the dominant surface.
- Compact or visually soften the top navbar and persona/action strip.
- Reduce visual weight of the left Visual panel during editing without hiding it.
- Keep all actions reachable.

3. Refine Content panel internals
- Slim down the source/persona row.
- Make Story Blocks metadata compact and quiet.
- Convert Add Block into a lightweight inline toolbar, not a heavy tray.
- Recast Compose with Sutradhar as a subtle assistant tray, not a full card.
- Make Tags clearly secondary.

4. Improve Save/Cancel integration
- The bottom Save/Cancel bar should feel like part of the editor surface, not pasted on.
- Reduce height, heavy border, and floating-strip feeling.
- Save should remain the primary action.
- Cancel should be quiet.

5. Reduce accent overuse
- Terracotta should be reserved for primary action, active state, and important highlights.
- Secondary actions like AI Assistant, Delete, utility buttons, inactive tabs, metadata, and helper tools should be quieter.
- Do not make the page colorful.

6. Preserve functionality
- Do not remove features.
- Do not change data flow or product logic.
- Prefer CSS/class/layout changes.
- Use existing isEditing state if available.
- Avoid risky logic changes.
