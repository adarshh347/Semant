# VISION-BUILD-001 · Increment B · B2 — General segmentation adapter — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-differential` · **Scope:** B2 only.
Continues from B0 (`ad5a1bd`, CUDA) and B1 (`7ea5aec`, orchestrator skeleton).

## What was built

The existing `yolo11n-seg.pt` path is now the general **`Segmenter`** adapter, and its
**native instance masks are the authoritative geometry**.

- **`vision_orchestrator/adapters.py`** — `masks_to_regions` (pure, torch-free helper):
  each instance's 0/1 mask → `mask_rle` (via `mask_geometry.rle_encode_mask`), then
  `canonicalize_geometry` derives `polygons`/legacy `polygon`/`box` and stamps lineage.
  Mask, class, confidence and model/preprocessing/device provenance are bound to the same
  candidate id. Empty instance masks are dropped — never coerced to a box, never
  fabricated. `YoloSegmenterAdapter` implements the orchestrator `Adapter` protocol
  (lazy ultralytics, GPU via `ModelManager`, load/unload/infer, returns regions).
- **`mask_geometry.rle_encode_mask`** — numpy fast path producing byte-identical COCO RLE
  to the pure encoder, so real image-resolution masks encode in O(runs) not O(h·w) Python.
- **`segmentation_service.segment_image_bytes`** now emits `mask_rle` (+ derived forms)
  through the shared helper, so `detect_regions` produces exact-mask regions with no route
  rewrite and legacy fields (`box`/`polygon`/`label`/`category`) preserved.
- **`routers/posts.py` geometry guard** — a mask-bearing region is **never** clipped or
  collapsed by semantic parenting; only a box-only fine part may be nudged into its
  parent. A canonicalisation pass stamps lineage on every assembled region without
  fabricating masks. The old label-parent/clip path can no longer mutate a mask.

## Architectural seams
- Masks enter through Increment A's canonical `mask_geometry` — **no second geometry
  contract**. Box is always derived from the mask; never the stored identity.
- `segmentation_service` (low-level, CPU today) and `YoloSegmenterAdapter` (orchestrated,
  GPU) share one conversion (`masks_to_regions`), so both paths yield identical geometry.
- `detect_regions` legacy response shape (`{regions, source, anchor_count, fine_count}`)
  is unchanged; consumers keep working, now with `mask_rle`/`polygons` added.

## Gate verification

- **Automated — 12 new tests, backend suite 81 passed:** numpy-encoder ≡ pure encoder
  (edge-touching incl.); multi-object collage stays N regions with distinct masks; a
  five-instance collage stays five regions on five slices; disconnected instance → ≥2
  polygon rings; edge-touching + portrait/landscape aspect; empty mask dropped (no
  fabrication); **mask geometry independent of any parent box** (never clipped/downgraded);
  YOLO-shaped region persists through the `Region` model + JSON; COCO→category mapping.
- **Real model evidence** (`B2-people-masks.png`, `B2-five_sculpt-masks.png`,
  `B2-*-alphacrop.png`) — YOLO on **CUDA through the orchestrator**:
  - group photo → **4 person instances**, each an exact silhouette mask; every region
    `box == rle_bbox_norm(mask_rle)`; each region **saved→reloaded** (Region model + JSON)
    with byte-identical mask; **rendered** as exact multi-ring outlines; **cropped** to an
    alpha identity crop (513×856).
  - five-sculpture → COCO-YOLO recalls **1** "person" (0.566); it keeps its exact mask.
    The regression's real protection — masks cannot be **collapsed or clipped by semantic
    parenting** — is proven by the five-region test and the parent-independence test.
    YOLO's low recall on sculpture heads is a model-quality matter (a domain segmenter /
    SAM refinement in B3+ addresses it), not a geometry collapse, and is reported honestly.
  - VRAM 63 MiB → 63 MiB after `unload` (clean release); group-photo inference ~2.9 s cold.

**Gate result: PASS** — fresh YOLO regions save/reload/render/crop as exact masks; the
five-sculpture cannot be collapsed or clipped by semantic parenting.

## Known limitations
- `segment_image_bytes` still runs YOLO on **CPU** (device policy for that legacy helper
  unchanged this increment); the orchestrated `YoloSegmenterAdapter` runs on GPU. Unifying
  detect-regions onto the orchestrated GPU adapter is a follow-up within B/E integration.
- COCO-YOLO recall on non-COCO subjects (sculpture, ornament) is inherently low — the
  reason the roster adds SAM2 refinement (B3) and domain parsers (C).
