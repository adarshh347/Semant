# VISION-D · D1 — Evidence packet — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-d-semantic`.

## What was built
`backend/services/evidence_packet.py` — `build_packet(post, image_bytes, intent)` assembles
one canonical VLM input from existing evidence:
- **whole image** (base64, ≤768px).
- **numbered mask contact sheet** — the image with each candidate mask **outlined** and a
  numbered badge at its centroid, so the VLM can see which candidate N refers to.
- **candidates** — per candidate: `n`, `candidate_id`, detector, category, `auto_label`,
  and — marked **AUTHORITATIVE** — the curator's label/note (creator-authored or noted).
- `allowed_ids`, the profile, model provenance, `image_hash`, and the compact **prompt**.
- **Stable numbering** — candidates are sorted by id, so number N always maps to the same
  `candidate_id` for the same region set → **survives retries + caching**.

The prompt binds interpretation to the supplied ids and forbids what the schema also
forbids: *"reference candidates ONLY by the ids above — never invent ids or move a label to
a distant candidate; output NO geometry; do NOT overwrite an AUTHORITATIVE curator label;
if the automatic candidates are insufficient, say so via needs_better_evidence."*

## Gate — candidate IDs map to the correct masks

- **Architecture** (`D1-contact-architecture.png`, Pietà): badge **1** on the wall, **2** on
  the floor, **3** on the Pietà figure — each numbered outline traces the right mask;
  `1=arch_0 wall`, `2=arch_1 floor`, `3=seg_0 person`.
- **Collage** (`D1-contact-collage.png`, five-sculpture): 10 candidates
  (`fine_0 Solanki face` … `fine_9 Gandhara crown`) each numbered on its (known-corrupted
  REGION-GEOMETRY-001) mask — the packet presents them **faithfully**; it does not fix or
  move geometry (not D's job). This is the D5 adversarial fixture: labels like
  "Solanki face" sit on candidate ids whose masks are piled near Gupta, so D5 can prove the
  VLM's labels stay bound to their ids and never re-parent geometry.
- **Automated:** 5 tests (`test_evidence_packet.py`) — structure + base64 decode; stable
  numbering (sorted by id, identical across reruns → cache/retry safe); curator labels
  marked authoritative + surfaced in the prompt; prompt binds to ids and forbids geometry;
  empty regions → empty candidates. Backend suite green (D0+D1: 25 tests).

**Gate result: PASS** — the packet maps stable candidate ids to the correct masks (shown for
architecture + collage), marks curator labels authoritative, and gives the VLM context
without permission to redefine geometry.
