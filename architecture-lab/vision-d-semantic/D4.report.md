# VISION-D · D4 — Real curator UX for the reading — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-d-semantic`.

## What was built

A **Read** tool in the Differential workspace turns the VLM reading (D0–D3, already
persisted on `post.semantics`) into a curator surface. The panel lives in the inspector
(`aside.diff-inspector`) beside the existing Select / Refine tools — the model interprets the
parts it was given; it never moves a mask.

- **`differential/useSemanticRead.js`** — the read/curate loop, cloned from `useMaskRefine`'s
  pattern: a `read({intent, force})` call to `POST /semantic-read` with an `AbortController`,
  a client timeout (70 s), request-sequence dropping of stale responses, and a state machine
  `idle | loading | ready | unavailable | timeout | error` mapped from the orchestrator job
  status. `curate(candidateId, patch)` calls `PATCH /semantics/:id` and folds the returned
  `semantics` back into state. `cancel()` aborts an in-flight read. It seeds from the post's
  existing `semantics`, so a prior reading shows instantly with no model call.
- **`differential/SemanticReading.jsx`** — the panel:
  - **Request / re-read** with an intent selector (name / material / relate / compose /
    describe), a `role="radiogroup"`.
  - Per assertion: label (curator override shown when set), **ranked alternatives** (click to
    adopt as the label), **confidence** bar, part/material/style/attributes facets,
    uncertainty, a **provenance** line (`provider · model`), and a status chip.
  - **Accept / Tentative / Reject / Edit** — the four curator decisions. Edit opens an inline
    field seeded from the curator's own text (never the model's), Enter saves as an override.
  - **Relations** — endpoint-citing proposals (`from → relation · confidence → to`), each
    endpoint a button that focuses that part on the image. They are read-only proposals: the
    panel never uses them to reparent or clip a mask.
  - **Whole-image reading** — composition / atmosphere / colour / scene / notes, visually
    separate from the object cards (it carries no candidate id).
  - **Needs better evidence** — when the model flags a candidate, a button hands the curator
    straight to the **Refine** tool on that part (`selectRegion` + switch tool).
  - **States** — loading (spinner + Cancel), unavailable (+ Retry), timeout (+ Try again),
    error (+ Retry), all under an `aria-live` region.
- **Backend enablers** — `_to_semantics` now surfaces `needs_better_evidence` (so the UX can
  launch Refine); `tentative` is a sticky curator state (`_STICKY_STATES`) so it survives a
  rerun like accept/reject/override.

## Gate — verified live in the browser (Chrome), on the Pietà

`localhost:5199` → post `695be77ea9ea58f1b6aef5eb` → Differential → **Read**:

1. **Panel renders the real reading** — three cards (wall 85 %, floor 80 %, person 90 %) each
   with ranked alternatives, facets (`part · sculpture`, `marble`, `Renaissance`…),
   uncertainty and `cloud_vlm · openai/gpt-4o-mini` provenance; two relations
   (`wall background-of person · 75 %`, `floor base-of person · 70 %`); a whole-image reading
   ("centralized… sacred and contemplative").
2. **Accept** on `person` → status `accepted` (green). **Edit** on `wall` → typed
   *"apse wall (curator)"* → status `overridden`, `curator_label` stored, the model's original
   `label:"wall"` **kept** (never overwritten).
3. **Live re-read** (a real second model call, `from_cache:false`, `status:succeeded`):
   - curator decisions **survived** — `wall` stayed `overridden` with the curator label,
     `person` stayed `accepted`; only the untouched `floor` (`proposed`) refreshed.
   - **geometry byte-identical** — `region_annotations` sha256
     `5d4046cb52c47715d6526c3c26e54e45` **unchanged** across the rerun. The reading writes
     only `post.semantics`.
4. **States** — the loading spinner + disabled intents + Cancel were observed during the live
   call; the panel returned to its ready state on completion.

Automated: two backend enabler tests (`needs_better_evidence` surfaced; `tentative` sticky) —
backend **151 passed**. Frontend `npm run build` clean.

**Gate result: PASS** — a curator can request a reading, weigh alternatives / confidence /
provenance, accept / edit / reject / hold-tentative, read relations and the whole image, and
send a poorly-evidenced part to Refine — all without the model ever touching a mask, and
without a rerun ever overwriting the curator's own decisions or the geometry.

## Notes
- The reading panel edits labels/status only; masks stay owned by the detector / SAM / Refine
  path (the schema is `extra="forbid"` end-to-end, so geometry is unrepresentable).
- The dev app ran on `:5199` (an unrelated app already held `:5173`); backend on `:8000`.
