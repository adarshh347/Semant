## What D3 formalizes

D2 already produced both scopes on the live Piet√† read; D3 pins them down as invariants and
proves the one thing that must never happen ‚Äî a relation turning into geometry.

- **Two distinct scopes.**
  - *Candidate-bound assertions* reference a supplied candidate id (`CandidateSemantics.candidate_id`)
    and are the only thing that speaks about a specific mask.
  - *Image-global reading* (`GlobalReading`: composition / atmosphere / colour / scene / notes)
    carries **no** `candidate_id` ‚Äî it reads the whole image and never pretends to be an object mask.
- **Relations stay proposals.** `RelationProposal` cites its endpoints (`from_id`, `to_id`),
  a free-text `relation`, and `confidence`. It is persisted under `post.semantics.relations`
  and can **never** set a region's `parent_id`, change mask ownership/containment, or clip a
  mask. Geometry lives only in `region_annotations` and this path never writes there.
- **Painting escape hatch.** `needs_better_evidence` (a list of candidate ids) lets the painting
  read declare that the automatic object masks are the wrong ontology ‚Üí Differential Grounds,
  rather than forcing a false object label. The evidence prompt offers this hatch explicitly.

Geometry remains structurally unrepresentable in every scope: `GlobalReading`,
`CandidateSemantics` and `RelationProposal` are all `extra="forbid"`, so a box/polygon/rle/
coordinate field raises at parse ‚Äî a relation or a "global read" cannot smuggle pixels.

## Gate ‚Äî relations cannot modify mask ownership/containment; scopes stay separate

Automated (`backend/tests/test_vision_d3.py`, 4 tests):

1. **Two scopes distinct** ‚Äî every stored assertion carries a `candidate_id`; the global
   reading carries none and round-trips (`composition == "central"`).
2. **Global forbids geometry** ‚Äî `GlobalReading(composition=‚Ä¶, box={‚Ä¶})` raises (`extra="forbid"`).
3. **Relation ‚â† parenting/clip** ‚Äî a VLM proposal "seg_0 part-of seg_1" is stored as a proposal
   (endpoints + type + `confidence == 0.7`); `region_annotations` is **byte-identical** before
   and after (`copy.deepcopy` equality) and **no** region gains a `parent_id`.
4. **Painting can declare candidates insufficient** ‚Äî `needs_better_evidence == ["seg_0"]`
   flows through, the global reading can say objects are "the wrong ontology", and the built
   evidence prompt contains `needs_better_evidence` (the hatch is offered to the VLM).

Backend suite: **149 passed** (145 + 4).

**Gate result: PASS** ‚Äî the two semantic scopes are structurally separate; relations are
endpoint-citing proposals that cannot become geometry parenting or containment and leave mask
bytes untouched; painting may declare the candidate set insufficient and point to Differential
Grounds instead of mislabelling.

## Note (test isolation)

These tests intentionally build identical evidence packets (same masks, same image) to probe
the invariants, so they share a content cache key. An autouse fixture clears the module-level
semantic cache between tests ‚Äî otherwise one test's cached response leaks into the next read.
This is a test-only concern; in production distinct posts/candidate sets yield distinct keys,
and `force:true` bypasses the cache while preserving prior curator state.
