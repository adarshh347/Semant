# VISION-C · C2 — Architecture profile — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-c-domain-profiles`. Integrates
`nvidia/segformer-b0-finetuned-ade-512-512` as the architecture specialist, through a
canonical adapter + the existing detect pipeline.

## What was built
- **`architecture_segmentation_service.py`** — SegFormer-B0 (ADE20K, 150 classes) →
  SEPARATE canonical mask candidates. Architectural ADE classes (wall, floor, ceiling,
  window, door, building, column, stairs, railing, …) selected by NAME → coarse category
  (surface / opening / structure); each connected component becomes its own
  `mask_rle` (via `mask_geometry`) with derived polygons/box + provenance. Friendly
  canonical labels (windowpane→window, etc.). **Labels never touch geometry** — the mask
  is the evidence; the ADE class is metadata only.
- **`SegFormerAdeAdapter`** (orchestrator) — capability `ARCH_PARSE`, resource **CPU**
  (measured fine), real (not deferred); load/unload/infer, returns arch-mask regions.
- **`detect_regions` wiring** — a `_profile_chosen(post)` helper reads the persisted
  domain profile; when `architecture` is chosen, the arch pass runs and is layered over
  YOLO **by geometry/precedence** (`merge_with_precedence`), never label-parenting.
  General/YOLO always runs; specialists add their masks beside it.

## Known limitations
- ADE b0 is a scene parser, not a facade specialist; fine ornament and dense window grids
  are coarse. The build contract's optional CMP-Facade checkpoint is a later refinement.
- The architecture pass runs when the profile includes `architecture`; auto-proposing the
  profile at detect time (rather than requiring a prior propose) is C5's planner wiring.
