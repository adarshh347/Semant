# VISION-C · C4 — Painting profile — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-c-domain-profiles`. Reuse-and-verify:
no universal painting segmentor is invented; the painting profile composes shipped
capability — SAM 2.1 Tiny (B5 Refine) for prompted individuation, optional SegFormer-B0
scene proposals, and Differential Grounds (Differential v1) as first-class non-object
evidence.

## The painting profile
- **SAM-prompted individuation is primary** (`passes_for(["general","painting"])` includes
  `sam21_hiera_tiny`); the general YOLO pass stays only as the cheap fallback; there is
  **no dedicated object parser forced onto a painting** (no fashionpedia/dense detector).
  `PROFILE_PASSES["painting"] = [sam21_hiera_tiny, segformer_b0_ade]` — SAM + optional
  scene proposals where meaningful.
- **Grounds are the non-object workflow.** Differential's Field · Path · Boundary ·
  Constellation · Relation · Frame (shipped, tested in `grounds.test.js` etc.) are the
  right ontology for light, dissolution, atmosphere and composition — evidence that is not
  an object mask. A painting may begin from the curator's gesture; nothing in the profile
  contract requires an auto object to exist (`test_painting_never_requires_auto_objects`).
- **"No useful automatic objects is not failure."** The contract never asserts regions
  must be produced; when object segmentation is the wrong ontology the curator uses Grounds.
  (The explicit UI copy for this lands with the C5 profile-control UX.)

## Gate — figurative + abstract: both object-mask and non-object workflows

- **Object-mask workflow (figurative):** on the Vishnu painting/lithograph
  (`695be7fa`), a **SAM box-prompt individuated the central figure** — score **0.786**, a
  49-point exact mask (`refine-region/preview`), through the normal refiner the painting
  profile schedules. Prompted individuation works on painterly/engraved content where an
  automatic object detector would flail.
- **Non-object workflow (abstract / light / dissolution):** Differential Grounds are
  first-class and object-independent — Brush (Soft Field) for where light lives, Trace
  (Boundary) for a dissolving seam, Frame for the whole composition. These are shipped
  Differential v1 and require no auto regions. (Browser-demonstrated end to end in the
  Differential v1 work; the painting profile simply doesn't crowd them out with forced
  object masks.)
- **Automated:** 3 tests (`test_vision_c4_painting.py`) — SAM primary / no forced object
  parser; painting passes reference only real adapters; the contract never requires auto
  objects. Backend **108 passed**.

**Gate result: PASS** — figurative individuation via SAM prompts and non-object Ground
workflows both work; painting is not forced into object categories, and an empty
auto-object result is a valid, non-failure outcome.

## Known limitations
- The FashionCLIP router underscores paintings unevenly (a monochrome lithograph scored
  `general`, not `painting`) — the curator override (C0) corrects it; a stronger
  painting/artwork classifier is a later refinement.
- The explicit "no objects here — start from a gesture" UI affordance is delivered with the
  C5 profile control.
