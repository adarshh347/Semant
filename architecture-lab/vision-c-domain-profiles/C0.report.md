## What was built (no specialist models yet — C2/C3 install those)

- **`services/domain_profiles.py`** — the persisted, multi-label, user-overridable
  contract. Profiles: `auto · general · fashion · architecture · painting`.
  `propose_profile(scores)` → general always + every specialist ≥ gate (0.35), with the
  `reason` (the scores) and the `scheduled_passes` it entails; `apply_override(...)` →
  the curator's choice wins, `user_overridden=True`, persists. `passes_for` maps chosen
  profiles → roster adapter names (`general` = yolo11n_seg + sam21_hiera_tiny always, as
  fallback not prison; fashion → fashionpedia_r50fpn, architecture → segformer_b0_ade,
  painting → sam + optional segformer).
- **`fashion_clip_service`** — added a `painting` prompt and `classify_domains(img)`
  returning multi-label confidence per specialist domain + a reason (reuses the existing
  zero-shot router; photography/food/product mass → general only).
- **Schema** — `domain_profile: Optional[dict]` on `Post`/`PostUpdate`, exposed via
  `post_helper`.
- **Endpoints** — `POST /{id}/domain-profile/propose` (auto-classify + persist, but never
  clobber an override) and `PATCH /{id}/domain-profile` (curator override).
