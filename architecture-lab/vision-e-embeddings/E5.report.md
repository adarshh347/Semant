# VISION-E · E5 — "Find similar" circulation — report

**Date:** 2026-07-19 · **Branch:** `feat/vision-e-embeddings`.

## What was built

A **Similar** tool in the Differential turns a selected confirmed Region into a visual-neighbour
query — presented as *inspectable research, never fact*.

- **Backend** (`backend/services/find_similar_service.py` + endpoints in `routers/posts.py`):
  - `POST /{post_id}/regions/{region_id}/find-similar` — routes the query to its space
    (DINOv2 identity/context, or FashionCLIP for a fashion domain), **indexes the Region on
    demand**, **re-indexes when its mask/source changed** (never on a label edit), searches ONE
    space, **excludes the Region itself**, and hydrates each neighbour with what the curator needs
    to judge it: source post, exact-mask geometry (for in-situ recall), similarity, model space
    and provenance. It creates **no Motifs or Relations**.
  - `GET /{post_id}/regions/{region_id}/crop?role=identity|context` — the exact-mask evidence crop
    (PNG) used for thumbnails and inspection.
  - `GET /vision/retrieval-spaces` — spaces + live availability.
  - Scope is own-corpus (every post already indexed in the query's model space) — never the whole
    world.
- **Frontend** (`differential/useFindSimilar.js` + `FindSimilar.jsx`, wired as the `similar`
  tool): identity/context toggle; result cards showing the evidence crop, similarity %, model
  space and provenance; **hover recalls the neighbour's exact mask in its own source image**
  (the crop swaps to the source photo with the mask outline drawn); **Open source** opens the
  post in a **new tab** so the workspace is never lost; a **re-index** control for stale evidence;
  loading/cancel, unavailable, empty, error/retry states under `aria-live`; and a standing
  disclaimer — *"Visual matches, not meanings."* Neighbours are never promoted to facts.
