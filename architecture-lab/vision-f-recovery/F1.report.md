## What was built

`backend/services/vision_recovery.py`:

- **`curator_identity_hash(post)`** — the invariant a restore must reproduce: each Region's id +
  authoritative geometry (mask_rle / polygons / polygon / box / geometry_rev) + curator fields
  (note / prioritised / weight / label / actor), plus semantic curator decisions and
  grounds/percepts. Independent of transient fields (descriptions, created_at).
- **`backup_scope(post_ids)`** — a hashed, read-only snapshot of each post's mutable fields
  (`region_annotations`, `semantics`, `grounds`, `percepts`, `domain_profile` + image refs) and
  every sidecar embedding row; `post_snapshot` records `identity_hash`, `full_hash`, schema.
- **`restore_post(post_id, backup)`** — **per-post** rollback: restores the post's mutable fields
  and re-materialises its embeddings from the backup, idempotently (deterministic keys + upsert),
  scoped to ONE post — never a global destructive rollback.
- **`Ledger`** — a file-backed migration ledger, one row per (post, job), states
  `planned → running → reviewed → committed / failed / rolled_back / skipped`, with history; a
  `committed` row is never re-run (idempotency), and `pending()` drives resume.
